1. ESPECIFICACIÓN FUNCIONAL
================================================================================

1.1 PROPÓSITO
-------------
El programa ZMIRO_REPROCESS_OO está diseñado para reprocesar facturas
de proveedores que generaron errores durante su contabilización en la
transacción MIRO. Utiliza un enfoque completamente orientado a objetos
para garantizar mantenibilidad y escalabilidad.

1.2 REQUISITOS DEL NEGRO
-------------------------
- Identificar facturas con errores de contabilización
- Permitir selección masiva de facturas para reprocesamiento
- Ejecutar reprocesamiento usando BAPIs estándar de SAP
- Generar log detallado del proceso
- Mantener integridad de datos durante el proceso

1.3 ALCANCE
----------
INCLUIDO:
- Reprocesamiento de facturas con estado de error
- Selección múltiple de facturas
- Use de BAPI_INCOMINGINVOICE_CREATE
- Generación automática de log de procesamiento
- Interfaz gráfica con ALV interactivo

NO INCLUIDO:
- Creación de nuevas facturas (solo reprocesamiento)
- Modificación manual de montos
- Integración con workflows
-Exportación de resultados
- Autorización por materia

1.4 FLUJO DEL PROCESO
--------------------
1. usuario selecciona criterios de búsqueda
2. programa obtiene facturas con errores pendientes
3. sistema muestra facturas en ALV interactivo
4. usuario selecciona facturas a reprocesar
5. programa ejecuta reprocesamiento vía BAPI
6. sistema muestra log de resultados
7. datos actualizados exitosamente

2.2 CLASES Y OBJETOS
--------------------

CLASE LCL_MIRO_REPROCESS:
---------------------------
Definición: Clase local para manejar reprocesamiento de facturas
Características: FINAL, Pública, name="lcl_miro_reprocess"

TIPOS DEFINIDOS:
--------------
TY_INVOICE_DATA: Estructura para datos de factura
   - BELNR: Documento de item (RSEG)
   - GJAHR: Año fiscal (RSEG)
   - BUKRS: Empresa (RBP)
   - BELNR_I: Documento de factura (RBP)
   - SAPNR: Referencia (RBP)
   - XBLNR: Número de referencia (RBP)
   - LIFNR: Proveedor (RBP)
   - WRBTR: Monto (RBP)

TT_INVOICE_DATA: Tabla de TY_INVOICE_DATA

MÉTODOS PÚBLICOS:
-----------------
CONSTRUCTOR():
   Propósito: Inicialización de instancia
   Parámetros: Ninguno
   Lógica: Setup inicial (vacío para futuras implementaciones)

GET_INVOICE_DATA():
   Propósito: Obtener facturas con errores del sistema
   Retorno: TT_INVOICE_DATA
   Lógica:
     - Selecciona facturas no contabilizadas o mayores a 30 días
     - JOINS RSEG y RBP (hasta 100 registros)
     - Filtra por provedor no contabilizado (XRECH = '')

PROCESS_INVOICE():
   Propósito: Reprocesar factura individual
   Importa: IS_KEY type TY_INVOICE_DATA
   Retorna: RV_SUCCESS type ABAP_BOOL
   Lógica:
     1. Obtiene items de factura de tabla RSEG
     2. Construye header para BAPI_INCOMINGINVOICE_CREATE
     3. Construye items de la factura
     4. Llama BAPI para crear/reprocesar
     5. Chequea errores en RETURN
     6. COMMIT o ROLLBACK según resultado

PROCESS_SELECTED_INVOICES():
   Propósito: Procesar múltiples facturas seleccionadas
   Importa: IT_DATA type TT_INVOICE_DATA
   Exporta: ET_LOG type STRING_TABLE
   Lógica:
     - Loop por cada factura seleccionada
     - Llama PROCESS_INVOICE() para cada una
     - Construye mensaje de éxito/error
     - Agrega mensaje a tabla de log

DISPLAY_INVOICES():
   Propósito: Mostrar facturas en ALV grid
   Importa: IT_DATA type TT_INVOICE_DATA
   Lógica:
     - Crea container si no existe
     - Crea ALV grid si no existe
     - Construye field catalogue
     - Construye layout
     - Muestra datos en grid

SHOW_LOG():
   Propósito: Mostrar log de procesamiento en pantalla
   Importa: ET_LOG type STRING_TABLE
   Lógica: Loop y WRITE de cada mensaje

MÉTODOS PRIVADOS:
----------------
BUILD_FIELDCAT():
   Propósito: Construir catálogo campos para ALV
   Campos mapeados:
     - BELNR, GJAHR, BUKRS, BELNR_I, LIFNR, WRBTR
   Configuración: Referencias estádard a tablas DDIC

BUILD_LAYOUT():
   Propósito: Configurar diseño de ALV
   Configuración:
     - ZEBRA = 'X' (líneas alternadas)
     - CWIDTH_OPT = 'X' (ancho óptimo)
     - SEL_MODE = 'A' (selección múltiple)

DATOS PRIVADOS:
--------------
GT_INVOICE_DATA: TT_INVOICE_DATA
GO_CONTAINER: Ref to CL_GUI_CUSTOM_CONTAINER
GO_ALV: Ref to CL_GUI_ALV_GRID
GT_FIELDCAT: LVC_T_FCAT
GS_LAYOUT: LVC_S_LAYO

2.3 MÉTODOS PRINCIPALES EN DETALLE
----------------------------------

PROCESS_INVOICE() - Implementación Técnica:
   Variables:
     LS_HEADER: BAPI_INCINV_CREATE_HEADER
     LT_ITEM: TABLE OF BAPI_INCINV_CREATE_ITEM
     LS_ITEM: BAPI_INCINV_CREATE_ITEM
     LT_RETURN: TABLE OF BAPIRET2
     LS_RETURN: BAPIRET2
     LV_INVOICE: BAPI_INCINV_FLD-INV_DOC_NO
     LV_FISCAL: BAPI_INCINV_FLD-FISC_YEAR
     LT_RSEG: TABLE OF RSEG
     LS_RSEG: RSEG

   Lógica de Header:
     - INVOICE_IND = 'X' (indicador factura)
     - Pasar número de documento y año fiscal existente
     - Fechas actuales para contabilización
     - Datos de empresa y proveedor

   Lógica de Items:
     - Por cada item en RSEG:
       * Pasar número de pedido (PO_NUMBER)
       * Item de pedido (PO_ITEM)
       * Cantidad y unidad
       * Impuesto y monto

   Llamada al BAPI:
     FUNCTION MODULE: BAPI_INCOMINGINVOICE_CREATE
     Success: BAPI_TRANSACTION_COMMIT WAIT
     Error: BAPI_TRANSACTION_ROLLBACK

4. MANUAL DE USUARIO
================================================================================

4.1 EJECUCIÓN DEL PROGRAMA
---------------------------
1. Ejecutar transacción SE38 o código de transacción personalizado
2. Ingresar nombre: ZMIRO_REPROCESS_OO
3. Presionar Ejecutar (F8)

4.2 PANTALLA DE SELECCIÓN
-------------------------

Document Number (BELNR):
   - Ingresar documento específico o rango
   - Rango ejemplo: 5100000000..5100009999
   - Múltiple selección con botón de flecha grande

Fiscal Year (GJAHR):
   - Ingresar año específico (ej: 2026)
   - Rango para múltiples años
   - Default: año fiscal actual

Tips de uso:
   - Use selection criteria específicas para mejor rendimiento
   - EVITE seleccionar too many records
   - Use ranges para búsquedas masivas eficientes

4.3 REPROCESAMIENTO DE FACTURAS
--------------------------------

DESPUÉS DE EJECUTAR:
1. Sistema busca facturas con errores pendientes
2. ALV grid muestra lista de facturas encontradas
3. Screen muestra hasta 100 registros máximos

SELECCIÓN:
   - Click en fila completa para seleccionar
   - Ctrl+Click para selección múltiple
   - Shift+Click para selección de rango
   - Click en column header para sort

REPROCESAMIENTO:
   1. Seleccionar facturas a procesar (mínimo 1)
   2. Click botón REPROCESS (o F8 si lo asigna)
   3. Sistema muestra popup de confirmación si se configura
   4. Procesamiento ocurre en background
   5. Sistema muestra log de resultados en pantalla

ESTADOS DE PROCESAMIENTO:
   - Success: "Invoice XXX processed successfully"
   - Error: "Failed to process invoice XXX"
   - Log completo: pantalla con todos los mensajes

4.4 VISUALIZACIÓN DE LOG
-------------------------
El log se presenta en formato simple:

Ejemplo de log exitoso:
Invoice 5100001234 processed successfully
Invoice 5100001235 processed successfully
Invoice 5100001236 processed successfully

Ejemplo con errores:
Failed to process invoice 5100001234
Invoice 5100001235 processed successfully
Failed to process invoice 5100001236

Para análisis detallado de errores:
1. Verificar logs de FI
2. Revisar BAPIRETURN2 messages

5.1 MANEJO DE ERRORES
---------------------

ERROR COMÚN: "No invoices found for reprocessing"
Causa: Sin facturas cumpliendo criterios
Solución:
   - Ajustar selection criteria
   - Verificar existe invoices en RBP con XRECH blank
   - Revisar fechas si filtering por antigüedad

ERROR: "Please select invoices for reprocessing"
Causa: Sin filas seleccionadas en ALV
Solución: Seleccionar al menos una fila antes de REPROCESS

ERROR BAPI: Business partner locked
Causa: Factura siendo procesada por otro usuario
Solución: Esperar y reintentar más tarde

ERROR BAPI: Invalid document number
Causa: Factura no existe o cancelada
Solución: Verificar RSEG/RBP tables para documento
